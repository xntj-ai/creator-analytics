数据复盘 · 静态 HTML 报告页

把多平台数据复盘,
落成一张可展示的报告页

creator-analytics 是一个 Claude Code / Agent 技能。数据算完之后,它产出一张自包含的静态 .html 报告页:PPVI 浅色风格,核心结论卡先行、热力总表、专题散点/条形、分平台迭代建议、可勾掉的执行 Checklist —— 纯 Python 标准库生成,图表内联 SVG,零 JS 库,浏览器直接打开。

复盘真正难的不是把页面画出来 —— 是让洞察别在呈现层掉价,有纪律的步骤别被跳过。多数分析最后只剩一墙对话和几张零散截图,判断和下一步会话一结束就蒸发。creator-analytics 的产物是一个可复审的成品:开篇就给判断、量级列按对数刻度上色(一个爆款不洗白整列)、异常如实标注,最后是一份你真能勾掉的清单。

工作流程

从各平台后台导出,到一张报告页,中间只有四步。

导出

平台数据导出

视频号 / 小红书 / 抖音 / B站 / YouTube 各平台后台导出明细(csv / xlsx)。

→
解析对齐

对齐成一份 JSON

一期内容一个实体,按平台分组指标,解析成 _<topic>_data.json —— 数字唯一来源。

→
分析

按 playbook 挖结论

量级与率分开读、漏斗定位、对照组坐实因果、相关性现算 —— 结论先行。

→
产出

静态 HTML 报告页

结论卡 + 热力总表 + 专题图 + 迭代面板 + Checklist,浏览器直接打开。

数字单一来源 —— 生成脚本只读解析后的 JSON 算数,不手填任何数字;footer 注明「由脚本从原始导出解析,未经人工转录」。

报告页结构

页面按固定顺序组装,没有对应数据的区块可删;结论卡与 footer 口径声明任何时候都保留。

核心结论卡 cards

开篇先给判断。4–6 张,gold 中性 / jade 坐实 / terra 校正。

热力总表 heat table

多实体 × 多指标。列内对数刻度上色,异常打徽标。

专题散点 / 条形 charts

两指标关系 + 阈值参考线,内联 SVG。异常点放大标 terra。

gold · 中性强调 jade · 正面 / 坐实 terra · 警示 / 校正 / P0 grey · 待核实(不臆断)

报告不止于诊断 —— 结论之后是分平台的动作面板,和一份可逐条勾掉的落地清单。

结论先行 异常如实标注 footer 口径声明

视觉为浅色 ppvi:克制灰阶基底、五级字号、玻璃磨砂卡片;assets/template.html 是唯一样式权威源 —— 本仓库就是这套样式的源头。

核心特性

内置洞察方法论

insight-playbook.md 的 0→9 分析路径:量级与率分开读、漏斗定位、对照组坐实因果、集中度检查、相关性现算、归因到可改的动作。

可展示页面骨架

结论卡 → 热力总表 → 专题图 → 迭代面板 → Checklist → footer 口径声明,顺序固定、按需取舍。

组件 helper 直接抄

热力上色、条形图、散点图、四语义徽标、迭代面板、Checklist 渲染器,生成脚本从 components.md 直接抄。

纪律即护栏

数字单一来源、无因异常标「待核实」不臆断、调研与实测分开、迭代建议每周期重新调研不沿用存量。

适用场景

适合

分析做完,要落成报告页

  • 多平台(视频号/小红书/抖音/B站/YouTube)数据复盘
  • 结论要可展示、可复用、有口径有动作
  • 跨平台复盘流程的「报告页」步骤
不适合

不是这些事

  • 博客文章页 / 发布文案(各走各的流程)
  • 原始数据解析本身(上游另写脚本)
  • 通用图表库 / 实时 BI 看板
