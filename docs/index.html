Data review · Static-HTML report page

Turn a multi-platform review
into a presentable report page

creator-analytics is a Claude Code / Agent skill. Once the numbers are crunched, it produces a self-contained static .html report page: PPVI light styling, finding cards first, a heat master table, topic scatter / bar charts, per-platform iteration advice, and a tickable execution checklist — generated with the Python standard library, charts inline SVG, zero JS libraries, opens straight in a browser.

The hard part of a review isn't drawing the page — it's keeping the insight from decaying at the presentation layer and the disciplined steps from being skipped. Most analyses end as a wall of chat and a few stray screenshots; the judgment and the next steps evaporate the moment the session closes. creator-analytics's output is a reviewable artifact: it leads with judgment, colors magnitude columns on a log scale (so one viral hit can't whitewash the rest), flags anomalies honestly, and ends with a checklist you can actually tick off.

Core features

Insight methodology baked in

The 0→9 analysis path in insight-playbook.md: read magnitude and rate apart, locate the funnel, nail causation with control groups, check concentration, compute correlations, attribute to a changeable action.

A presentable page skeleton

Finding cards → heat table → topic charts → iteration panel → checklist → footer cadence statement, fixed order, drop what you don't have.

Copy-ready component helpers

Heat coloring, bar chart, scatter plot, four-semantic badges, iteration panel, checklist renderer — the generation script copies them straight from components.md.

Discipline as guardrails

Single source of numbers, anomalies with no cause marked "to verify" not guessed, research kept apart from measured data, iteration advice re-researched each cycle.

When to use

Good fit

Analysis done, needs a report page

  • Multi-platform (Channels / Xiaohongshu / Douyin / Bilibili / YouTube) data reviews
  • Conclusions that must be presentable, reusable, with cadence and actions
  • The "report-page" step of a cross-platform review
Not for

Not these jobs

  • Blog-article pages / publish copy (each has its own flow)
  • Raw data parsing itself (an upstream script handles that)
  • A generic charting library / real-time BI dashboard
