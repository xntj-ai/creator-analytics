# 自媒体数据分析 → PPVI 浅色报告页

把分析结论落成一张可公开展示的 HTML 报告页。零依赖：纯 Python 标准库生成静态 HTML，图表用内联 SVG，不引任何 JS 库。

## 铁律

1. **数字单一来源** — 先把各平台后台原始导出解析成一份对齐的 `_<topic>_data.json`，报告生成脚本只读 JSON 算数渲染，不手填任何数字；footer 注明「由 <脚本> 从原始导出解析，未经人工转录」。
2. **结论先行** — 开篇就是「N 条核心结论」卡片，先给判断再展开；每个专题章节的开头段也是结论先行。
3. **审查纪律** — 异常数据（如 0 播放）无可见原因标「待核实」（灰徽标）不臆断原因；外部调研类论断标「(调研)」与本账号实测区分；比率一律用 %（9.8%）不用小数；被推翻过的旧结论标「校正」。
4. **口径有记忆** — 维护一份「已坐实结论」口径文件，写结论前先读；别拿单期数据轻率推翻多周期验证过的结论，有冲突先说明再改口径。
5. **公开/私有分版** — 报告页默认按「会公开展示」对待：变现策略（承接钩子/漏斗/定价）不进公开页，需要时另出私有文件。

## 分析纪律（报告的全面性来自这里，不是模板）

- **迭代建议不沿用上期存量**：每个周期按需起并发调研智能体（每平台一个）刷新当期平台打法，结合本期数据重写；上期建议只用于「执行了没有、见效没有」的对照。
- **新专题现算现验**：相关性（如 收藏率×播放 的 Pearson）、归因验证（如 逐条核对内容开场与完播的关系）每期现算，不套旧结论。
- 完成后抽查 3-5 个页面数字回 JSON 核对。

## 工作流

1. **输入**：对齐后的数据 JSON + 本轮分析结论。对话中被用户校正过的口径优先于任何旧结论。
2. **建生成脚本**：`build_<topic>_html.py`（建议 PEP 723 头，`uv run --script` 跑，deps 留空）。整套 CSS 和章节骨架从 [assets/template.html](assets/template.html) 取——它是唯一样式权威源，别即兴改 token；放进 f-string 时 CSS 花括号 `{` 要写成 `{{`。
3. **组装章节**：按下方页面结构取舍；热力表/条形图/散点/徽标/迭代面板/Checklist 的 helper 代码直接抄 [references/components.md](references/components.md)。
4. **输出**：静态 HTML，本地浏览器直接打开即可，无需服务。

## 页面结构（按需取舍，顺序不变）

```
kicker + h1(标题+区间) + lead(北极星/本次要回答的核心问题)
→ stat strip（3-4 个全局汇总数）
→ N 条核心结论卡（4-6 张，全页判断的浓缩）
→ 汇总热力总表 + legend + note（口径说明）
→ 专题章节 × N（h2 一句话立场 + lead 结论先行 + 图/表 + two-panel 解读）
→ 迭代建议面板（分平台多维动作，P0 标红）
→ ★ 战略结论卡（本轮校正后的关键修正）
→ 候选/下一步表（如有）
→ 执行 Checklist（可勾掉的动作清单）
→ footer（数据来源 + 审查纪律声明 + 生成脚本名）
```

没有跨期/跨实体对照就删总表，没有落地动作就删 Checklist；**「核心结论卡开篇」和「footer 口径声明」任何时候都保留**。

## 组件选型

| 数据形态 | 组件 |
|---|---|
| 3-4 个全局汇总数 | stat strip |
| 一句话判断 + 证据 | card（gold 中性 / j=jade 正面坐实 / t=terra 警示校正）|
| 多实体 × 多指标横向对照 | 热力总表（列内 log 刻度上色 + 异常徽标）|
| 单指标 TopN | bars 条形图 |
| 两指标关系 / 找异常点 | SVG 散点 + 阈值参考线 |
| 成对解读（真因/对策、正面/反面）| two-panel（j/t 配对）|
| 分平台多维建议 | iter 面板（⚑→P0）|
| 落地动作 | checklist（P0 优先级）|

色彩语义贯穿全页：**gold #C8922E** 中性强调 · **jade #1f6f5c** 正面/坐实 · **terra #B4502E** 警示/校正/P0。
